## 背景
我们正在开发 demo 以便展示 sched 如何使用，目前我搭了一个基本的示例。
## 思路
我的想法是我先大体设计一下我们的战斗框架，然后你分析需求实现 Unit 框架，然后我们再填充技能
## 需求
- demo 的设计必须适配 sched (@sched/integration.md)
- 框架设计和测试代码需要分开不同的文件，避免过于混乱，框架设计也需要注意合理的文件设计
- demo 会创建 n x n 个 unit 形成一个正方形，相邻 unit 的间距是 1.0
- (x + y) % 2 相同的 unit 是同阵营，不同的是敌对阵营
- unit 在空闲时会随机选择一个攻击范围内（攻击范围5.1） 的敌人进行普通攻击，并设置索敌 5 秒（即5秒后再次进入空闲状态）；unit 攻击分为攻击前摇和攻击，攻击前摇阶段可以打断（后续解释什么行为可以打断），攻击行为产生后进入攻击cd，攻击会延迟造成伤害（模拟弹道时间），时间基于距离/弹道速度
- 游戏中所有cd/时间单位为秒，每个tick设计为0.125s，即一秒模拟8个tick，模拟并不根据真实事件计算，而是每个tick给world时间增加 0.125s，游戏中的cd/弹道时间虽然是连续时间，但会发生在tick上，即时间戳超过tick时间的那个tick发生，计算cd时却要从时间戳开始计算，避免累积误差
- 普通攻击/技能/buff 各自维护 think delay，并取最小值返回给 Think
- unit 最多可以挂载8个技能槽，每个技能技能槽支持挂载一个技能（允许重复）。技能的状态包括未就绪/就绪/施法前摇/施法后摇，技能就绪后会在 unit 施法队列中排队，unit当前普通攻击结束后依次执行排队就绪的技能，就绪的技能并不一定能够释放（例如禁止技能buff/打断/找不到目标/条件不足），此时施法会被丢弃（技能进入OnFail），施法队列排空后才能重新进入普通行为。
- 技能目前有多种就绪方式，主要是cd和被动技能，cd型技能当cd到期后即可以进入就绪，释放后立即进入cd；被动技能分为两种，纯数值影响型和释放型，释放型被动技能则在收到事件触发(主动攻击/收到攻击/释放技能[技能联动])时Think，可以直接让配置的技能进入就绪，也可以配置条件，如概率，充能等
- unit 血量降低到0后会进入死亡状态，经过默认8s(有一些技能可以加速该时间)后满血满蓝复活
- 攻击范围/攻击力/攻击速度/血量上限都存在技能/buff/debuff可以影响
